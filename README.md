# twya.vanilla.js

**tw.vanilla.js** (Tailwind Vanilla JS) — це мій навчальний репозиторій-майстерня, де я крок за кроком прокачую **Vanilla JavaScript** на практиці та збираю невеликі UI-компоненти, використовуючи **Tailwind CSS** для стилізації.

Ціль проста: не “просто дивитися туторіали”, а **закріплювати знання через маленькі, зрозумілі та повторювані приклади**. Кожен приклад — як окремий “модуль” тренування: мінімум магії, максимум логіки.  
Це місце, де помилки не соромно робити, бо саме вони і є паливом для прогресу 🚀

---

## 🎯 Навіщо існує цей репозиторій

- **Зберігати напрацювання** та не втрачати результат між сесіями навчання.
- Будувати **базу готових прикладів**, до яких можна повертатися.
- Вчитися писати **читабельний JS**, працювати з DOM та подіями.
- Закріплювати розуміння UI-поведінки: відкриття/закриття, стани, фокус, взаємодія з бекдропом тощо.
- Тренувати **структурування проєкту** та оформлення документації (README).

---

## 🧩 Що тут буде (або вже є)

Репозиторій складається з невеликих прикладів/компонентів на чистому JavaScript, наприклад:
- [модальні вікна (modal)](components/README.md)
- таби (tabs)
- акордеони (accordion)
- дропдауни / меню
- тултіпи
- слайдери (простий рівень)
- форми та валідація (база)
- дрібні інтерактивні штуки для практики DOM

Кожен приклад задуманий так, щоб його можна було:
- **швидко запустити**
- **прочитати як конспект**
- **розширити** під складніші потреби

---

## 🛠️ Технології

- **Vanilla JavaScript** — логіка та взаємодія з DOM
- **Tailwind CSS** — швидка стилізація та базова UI-естетика
- **HTML** — проста семантична структура

Цей проєкт навмисно без фреймворків: задача — навчитись “їздити”, а не одразу будувати спорткар із автопілотом 😄

---

## ✅ Принципи (щоб не перетворилось на хаос)

- **Один приклад = одна ідея.**  
  Краще маленький, але зрозумілий модуль, ніж “все одразу”.
- **Читабельність важливіша за трюки.**  
  Якщо рішення виглядає як магічний фокус, значить його треба спростити.
- **Пояснення важливіші за обсяг коду.**  
  README та коментарі (де доречно) допомагають мені самому через місяць.

## 🧭 Як користуватися

1. Обери приклад/компонент у репозиторії.
2. Відкрий відповідний файл(и) та подивись розмітку й логіку.
3. Зміни щось маленьке (текст, клас, поведінку) і перевір результат.
4. Поступово ускладнюй: додай закриття по `Esc`, фокус-трап, анімації, повторне використання тощо.
